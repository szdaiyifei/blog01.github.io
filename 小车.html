（2）下载对应python版本的Torch.whl文件

https://forums.developer.nvidia.com/t/pytorch-for-jetson/72048

a. 选择aarch64版本的whl文件

AArch64 是什么_会飞的金色流年的博客-CSDN博客_aarch64
ubuntu18.04自带的python是2.7和3.6.9，所以下载cpu/torch-1.9.0-cp36-cp36m-linux_aarch64.whl 版本
cp表示python的版本 cu表示CUDA的版本
同时需要到官方网站中去找需要的torchversion和torchaudio。如果无法访问外网，去清华源找也是可以的

b.安装torch

# 提前下载 torch需要的安装包
sudo apt-get install python3-pip libopenblas-base libopenmpi-dev 
pip3 install Cython
# 安装torch之前要先安装numpy
pip3 install numpy torch-1.9.0-cp36-cp36m-linux_aarch64.whl

c.安装torchvision和torchaudio

下载torchvision时会报如下错误，主要错误是相应的包的名字是manylinux2014 他要求pip的版本是>=19.3，具体教程可以参看 pip安装包属性含义 解决教程看(1)的e.
# 加--user  避免出现权限问题导致下图报错
# 文件名不得修改 torchvision-0.10.0-cp36-cp36m-manylinux2014_aarch64(1).whl 这个样子是错误的，会报is not supported wheel on this platform的错误
pip3 install  install  torchvision-0.10.0-cp36-cp36m-manylinux2014_aarch64.whl --user
pip3 install  install torchaudio-0.9.0-cp36-cp36m-manylinux2014_aarch64.whl --user

d.验证

python3

>>> import torch
>>> print('CUDA availabel: ' + str(torch.cuda.is_available()))
CUDA availabel: True
是true就是正确的。 如果是false就可能是torch版本不对应，或着cuda没写入环境变量等等需要一一排查
